 Styling Inert Elements Inert elements are also removed from the tab order and accessibility tree. So it’s the semantically correct way to signal that the user shouldn’t interact with certain parts of the page. Styling is easy, target inert in CSS and, for example, reduce opacity on inactive content. Controlling How Dialogs Close How to close it again? Three options, controlled by the closedby attribute:

This can only be closed with an explicit close button

Press ESC or click the button to close

Click anywhere outside to close

 If you set it to none, the dialog can only be closed via code, meaning through a call to close(). For example, triggered by clicking an explicit X or close button. closedby="closerequest" supports, in addition to an explicit close call which still works, platform-specific actions like pressing the escape key. And the third variant is closedby="any". That's the most extensive one. It additionally enables light dismiss, meaning the dialog closes when the user clicks anywhere outside of it. So, thinking back to the days when you would take a div, style it like a modal, then place another element behind it with fixed positioning across the entire page so it acts like a backdrop, then add a click event that hides the backdrop and the modal… it was a lot of work. Today you can do all of that with just a few lines of HTML and JavaScript. Native, clean, simple.
