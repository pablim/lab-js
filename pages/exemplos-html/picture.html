Using Picture for Responsive Images So, what can you do? Use the picture element. It allows you to serve different versions of an image depending on things like device size, orientation, dark mode, and more. How does it work? Start with your regular image element and a source path. Wrap the whole thing in a picture element. This already works, but doesn’t make much sense yet, it’s only one version of the image. But this image inside a picture is the fallback image, the one the browser uses when nothing else matches. Now you can add additional images for specific situations. For this, create a source element, put the URL of the, for example, smaller image in the srcset, and define a media query in the media attribute. Important detail: With multiple source elements, the browser picks the first one that matches. So if you define up to 500 pixels first, then up to 1,000 pixels, everything is fine. But if you swapped them, a tiny screen would still be smaller than 1,000 pixels, meaning the browser would load the medium image immediately and never reach the small one. So order matters here. Art Direction: Different Crops for Different Screens Another example I really like is using different images depending on orientation: In the source element, we set the media condition to orientation: portrait. And now in portrait mode, the browser loads a vertical image where the crop fits perfectly. Super practical, and it even updates dynamically when you resize the browser. Handling Pixel Density (Retina Displays) What we can also do with a picture element or a source element is target different pixel densities. A modern smartphone or laptop can display images much sharper and comes with a higher pixel density. You can group different pixel densities inside a single source element. Here, besides the fallback image, we have a variant with density 1x. You can actually omit the 1x because that’s the default. Then there’s a 2x version, simply an image that’s twice as large but displayed at the same size, so effectively twice as sharp. And then a 3x version. Width Descriptors Instead of pixel density, you could also use a width descriptor: The sizes attribute tells the browser how much space the image will take up at different viewport sizes, and the browser picks the most appropriate image from the srcset.
